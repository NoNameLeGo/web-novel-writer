# AI 网文写作工作流

## 概述

这是一个面向**长篇网络小说**的 AI 辅助创作工作流。它不是一个"你写一句 AI 补一句"的聊天模式，而是一套结构化的**导演式生产管线**：

> 灵感 → 题材定位 → 世界观搭建 → 人物设定 → 大纲规划 → 风格定义 → 章节生成 → 一致性检查 → 去AI味优化 → 导出

**核心理念**：AI 是导演和编剧助理，你是总编审。AI 负责结构规划、批量生成、一致性追踪，你负责创意决策和品质把关。

## 1. 通用工作流

### 阶段一：题材脑暴与定位

在与用户的对话中，完成以下结构化输出：

```
【题材定位卡】
- 书名（暂定）：
- 题材/标签： 玄幻/都市/科幻/言情/悬疑...
- 核心梗（一句话）：
- 目标读者画像：
- 竞品参考书：
- 卖点/差异化：
- 预计总字数：
- 更新频率：
```

### 阶段二：世界观搭建（世界书）

为小说创建结构化世界观档案。包含但不限于：

```
【世界观设定】
- 时代背景：古代/未来/异世界/架空...
- 力量体系（玄幻/修仙）：境界划分、修炼方式、特殊能力
- 科技水平（科幻）：科技树、核心设定
- 社会结构：权力格局、阶级、势力分布
- 地理/地图：主要地域、特殊地点
- 历史脉络：关键历史事件、传说
- 特殊规则：魔法/系统/规则类设定
```

**方法**：使用分步追问法迭代世界观。每轮只深挖一个维度，不要一次性全抛。

### 阶段三：人物设定

使用"角色卡"结构输出每个主要角色：

```
【角色卡】
- 姓名：
- 身份/标签：
- 外貌特征：
- 性格（MBTI+核心特质）：
- 背景故事：
- 能力/金手指：
- 动机/目标：
- 成长弧线（起始→终点）：
- 关键关系：
- 说话风格/口癖：
```

对于网文，建议至少设计：
- **主角**：1-2人，必须有清晰的成长弧线和核心动机
- **重要配角**：3-5人，每个有独立的叙事功能
- **反派**：1-2人，动机不能是"单纯的坏"

### 阶段四：大纲与节奏规划

使用**网文节奏模型**来规划篇章结构。推荐"三幕五卷"或"打怪升级"结构：

```
【全书大纲】
- 第一卷：【卷名】——核心事件、字数、起止章节
   - 主线：_________
   - 支线：_________
   - 爽点/高潮：_________
   - 结尾钩子：_________
- 第二卷：【卷名】——...
...

【章节级细纲】（以第X卷为例）
第1章：_________
  - 开场钩子：_________
  - 核心情节：_________
  - 出场角色：_________
  - 章尾钩子：_________
```

**网文节奏要点**：
- 每章至少1个小**爽点**（打脸/升级/收获/反转）
- 每3-5章1个**中高潮**
- 每卷1个**大高潮**
- 章尾必须有**钩子**（悬念/预告/反转）

## 2. 章节生成 Prompt 体系

这是整个 skill 的核心资产。以下是一套经过验证的多级 Prompt 模板。

### 2.1 章纲生成 Prompt

把规划阶段的大纲细化成可执行章纲：

```
你是一名资深网文作家，擅长[题材]类型。请根据以下信息生成第[N]章的详细章纲。

【全书设定概要】
- 世界观：{世界观简述}
- 力量体系：{核心规则}

【当前卷概况】
- 卷名：{卷名}
- 本卷核心冲突：{冲突描述}

【上一章回顾】
- 上章内容：{上章概要}
- 上章结尾状态：{上章结尾}

【本章要求】
- 本章目标：{推进什么情节/达成什么效果}
- 字数：{2000-3000}
- 必须包含：{具体要素}
- 章尾钩子类型：{悬念/反转/预告}

请输出章纲格式：
【章纲】
- 开场（100-200字）：
- 发展（主体情节）：
- 高潮/爽点：
- 收尾+钩子：
- 本章关键词：
```

### 2.2 正文生成 Prompt

```
你是一名网文作家，现在需要根据章纲写出正文。

【写作要求】
- 风格：{风格描述，如"轻松日常"、"热血战斗"、"悬疑推理"}
- 视角：{第一人称/第三人称}
- 时态：一般现在时
- 对话比例：约30%
- 描写比例：约20%
- 每段不超过200字
- 避免：大段景物描写、哲学议论、视角跳跃

【核心设定】（注入关键设定，保持一致性）
{世界观和力量体系的当前相关状态}

【本章章纲】
{章纲内容}

【输出要求】
- 直接输出正文，不要附加说明
- 每写完一章，标注章节标题
- 如果生成中断，标记中断位置

现在开始写正文：
```

### 2.3 续写/扩写 Prompt

```
请基于以下已有正文继续续写，保持以下要素一致：
1. 文风：保持已有的语言风格、句式节奏
2. 角色性格：{角色名}的性格特征是{性格描述}，言行必须符合
3. 当前情节走向：{情节方向}
4. 字数要求：{目标字数}

已有正文：
{已有正文内容}

请从标记处继续：
```

### 2.4 风格微调 Prompt（去AI味专用）

```
请对以下文字进行润色，目标：消除AI生成痕迹，使其具有人类网文作者风格。

修改要求：
1. 减少"然而"、"但是"、"因此"、"毕竟"等逻辑连接词的使用
2. 增加短句和口语化表达
3. 减少形容词堆砌，多用动词驱动叙事
4. 对话更自然，加入语气词和省略
5. 避免总分总结构，让情节自然推进
6. 减少"他/她/它"的指代，适当使用名字重复
7. 段落之间不要强行过渡

原文：
{待润色文本}
```

## 3. 写作管线（串联所有阶段）

当用户有明确需求时，按以下管线串联工作：

```
Step 1: 采集题材 → 输出【题材定位卡】
Step 2: 深挖世界观（3-5轮追问） → 输出【世界观设定】
Step 3: 创建角色组（主角+配角+反派） → 输出【角色卡集】
Step 4: 规划全书大纲 → 输出【全书大纲】
Step 5: 定义写作风格 → 输出【风格配置文件】
Step 6: 逐卷生成章纲 → 输出【章纲序列】
Step 7: 逐章生成正文 → 输出【正文】
Step 8: 一致性检查 → 角色/设定/剧情线是否跑偏
Step 9: 去AI味润色 → 输出【最终版本】
Step 10: 导出 → 纯文本/Markdown/DOCX格式
```

## 4. API 配置指南

本 skill 通过 OpenCode 环境变量或对话中直接指定来配置模型。

### 支持的API

| 提供商 | 配置方式 | 推荐模型 |
|--------|---------|---------|
| **OpenAI** | `api:openai` 或环境变量 `OPENAI_API_KEY` | GPT-4o / GPT-4o-mini |
| **Claude (Anthropic)** | `api:claude` 或环境变量 `ANTHROPIC_API_KEY` | Claude Sonnet 4 |
| **DeepSeek** | `api:deepseek` 或环境变量 `DEEPSEEK_API_KEY` | DeepSeek-V3 / DeepSeek-R1 |
| **OpenRouter** | `api:openrouter` 或环境变量 `OPENROUTER_API_KEY` | 聚合多种模型 |
| **Ollama (本地)** | `api:ollama` | Qwen2.5 / Llama3 / DeepSeek |
| **自定义 OpenAI 兼容** | `api:custom + CUSTOM_API_URL` | 任意兼容端点 |

### 模型选择建议

| 任务 | 推荐模型 | 理由 |
|------|---------|------|
| 世界观构建 / 大纲规划 | **DeepSeek-R1** / **Claude Sonnet** | 逻辑推理强，长上下文 |
| 正文生成（量大） | **DeepSeek-V3** / **GPT-4o-mini** | 性价比高，速度快 |
| 正文生成（品质优先） | **Claude Sonnet** | 文风最优，自然度高 |
| 去AI味润色 | **Claude Sonnet** / **DeepSeek** | 风格控制好 |
| 一致性检查 | **Claude Sonnet** / **GPT-4o** | 分析能力强 |
| 本地部署 | **Qwen2.5-14B** (Ollama) | 中文网文能力够用 |

### 在对话中切换模型

```
使用 DeepSeek 生成第5章正文
切换到 Claude 做润色
用 Ollama 本地模型跑批量续写
```

## 5. 检查清单（用于质量控制）

每次生成后，逐项核对：

### 角色一致性
- [ ] 角色名字是否写错
- [ ] 角色性格是否与前文一致
- [ ] 角色能力/境界是否与设定一致
- [ ] 角色关系是否与设定一致

### 情节一致性
- [ ] 当前情节是否与大纲一致
- [ ] 伏笔是否有照应
- [ ] 时间线是否有矛盾
- [ ] 地点/空间是否有逻辑错误

### 文风一致性
- [ ] 是否保持了前文的语言风格
- [ ] 对话风格是否符合角色设定
- [ ] 是否有突兀的AI表达（过于书面/逻辑词过多）

### 网文要素
- [ ] 本章是否有爽点
- [ ] 章尾是否有钩子
- [ ] 节奏是否太慢/太快
- [ ] 对话是否自然
